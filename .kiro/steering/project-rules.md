# Правила проекта Qogita

## Документация

- НЕ создавать новые markdown файлы для документации без явного запроса пользователя
- Вся документация должна быть в README.md
- Избегать дублирования информации
- Быть лаконичным в ответах и резюме

- Не создавать файлы "на будущее" без запроса
